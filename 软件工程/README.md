## 软件工程

**授课教师**：王鸽

**课程内容**：课程主要围绕软件开发完整生命周期的全过程来讲，包括项目立项、需求分析、设计、编码、测试、运行与维护，一共有12个章节。平时需要完成思源学堂上的课后练习。此外，还需要6人以内组队完成一个题目自选的项目，这个项目你需要完成立项报告、需求分析报告、设计报告，以及在课程结束时完成答辩。

**成绩组成**：10%平时成绩（思源学堂课后练习+考勤）、20%项目（三次报告占10%，答辩占10%）、70%期末。

**课程评价及经验**：

如果从就业的角度来看，这门课学习的东西其实还是有一些用处的。课程总体偏文科一点，需要背诵记忆的东西不少。

对于项目，我的评价是非常之水，实际上对于项目的要求大概是“完成即可”。最后的答辩也是抽签上去展示，没被抽到的（大多数组）录答辩的视频即可。

对于期末考试，难度也不高。请务必仔细阅读老师提供的复习PPT！因为课件是中英双语的，复习起来看着费劲也抓不住重点，我的建议就是纯粹面向复习PPT进行复习。只要掌握复习PPT里的所有知识，就能掌握考试考的全部内容。另：相当一部分的考试题目源于复习PPT，所以请务必仔细阅读！

此外，我可以提供一些期末考试考察的题型与题目：

选择题：20分，只要学习过复习PPT就没问题；

简答题：20分，一共有四道题，我尚且有印象的题目有：

1.内聚与耦合的定义？区别？

2.结构化开发的定义？结构化开发的三种程序类型？

3.说出一种顺序开发模型和一种迭代开发模型，并对你说出的模型进行讲解。

应用题：60分。

1.一个较为复杂的校园电话号码的情景题，要求给出电话号码的数据字典。

2.根据伪代码画PAD图。

3.给出一个场景，要求画出类图，不是很复杂。

4.关于测试的一系列题目，包括：画出程序的有向图，计算测试用例上限，用路径测试法设计测试用例。

总体课程比较简单，王鸽老师人也很好。
