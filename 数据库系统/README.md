## 数据库系统

**授课教师**：何亮

**课程内容**：授课内容主要有数据库基础知识、关系代数、SQL、数据依赖与关系模式规范、数据库设计、查询优化、事务管理、数据库安全这些。实验内容有两个可选，一个是读源码（可以选UNION、JOIN等多个主题，需要完成报告+一个20min的Pre），一个是正常的实验内容（在仓库中有指导书）。

**成绩组成**：10%平时成绩（考勤，作业）、30%实验、60%期末。

**课程评价及经验**：

何亮老师非常有意思，讲课喜欢结合工程实践，是我为数不多能上课听下去的课。上课会有一些神秘小活动，比如雨课堂回答问题（不查人，答对问题一次作业算满分），还有取作业（没有领取作业你可以选择：1.按缺勤处理，2.课前表演节目，3.于早上七点前往钱图合影发给他）。

与之相应的，何亮老师对学习这门课的要求也颇高。具体体现在期末考试有大约10分的题目是课程之外的内容，需要你平时了解一些课外的内容。

期末考试整体难度不小，部分回忆如下：

**选择题**：20题，每题一分。有难题也有简单题，印象比较深的题目有：

1.嵌入式SQL的游标作用是什么；

2.关系代数与关系演算产生的无限关系和无穷验证问题；

3.强制存取控制MAC的读写规则；

4.有一道题涉及到NoSQL的BASE特性；

5.在集合和包（允许重复的集合）上分别能否建立索引加快数据读取；

6.什么关系代数运算不是单调的（单调：指增加参与运算的元组个数，得到的结果数量不会变少）；

7.有一道题考察了Undo操作和Redo操作，问在两种不同规则（是否允许事务完成到一半就写入磁盘？是否允许事务已经完成了暂存于内存之后在写入磁盘？）的情况下，数据库恢复是否需要执行Undo和Redo操作，这里需要理解这两种操作才能做出题目；

8.意向锁的相容矩阵，考的是SIX锁和IS，IX，SIX锁的相容关系；

9.关于检查点的题目，考了什么时候开始/commit的事务需要Undo/Redo。

**大题**：

1.关系代数相关的大题，考了除操作，考了关系演算。

2.SQL查询相关的大题，一共5小问一道4分，前三题简单，第四题需要有一个复杂一点的嵌套查询，题目印象中是（每本图书有不同的经销商，查询最高销售额的经销商的图书的属性），第五题考了Update操作。总体来说，SQL只需要学会SELECT、UPDATE、DELETE、INSERT以及相关的一系列谓词即可。

3.查询优化相关的大题，首先让你画出一个关系代数优化后的语法树，大致是$\Pi_{abc}(\sigma_{abc}(C\times SC\times S))$这样的形式，和课件上一个例题考的差不多。还有一小问考到了设计合适的物理优化策略。

4.一些简答题，每道四分，题目有：

(1) 数据库存储的时候按页存储，一个页为什么在头部和尾部都要有校验位；

(2) 数据库恢复的时候，有两种日志记录方式，一种是逻辑日志（记录操作），一种是物理日志（记录具体值的变化），如果追求恢复的速度，应该用哪种；

(3) 三个事务并发运行，写出2pl和wound-wait两种策略的执行过程。

5.关于关系依赖与关系模式规范的题目：

(1) 对一个函数依赖添加一些依赖转为BCNF（$R(A,B,C),\{A\to B,B\to C\}$），把一个转为3NF（忘了，但是简单）；

(2) 求一个函数依赖集的最小函数依赖集；

(3) 把一个关系模式拆解为3NF。

6.ER图设计+拆解为关系模式，题目难度远高于作业题的难度。最后大概有6-7个关系模式。关于ER图设计的题目，无需考虑弱实体、部分参与等情况。

在我看来这无疑是一套好卷子，想拿高分难，想及格不算难。可以看到何亮老师想要重点考察理解能力，实际上这门课要背诵的东西确实不多。
