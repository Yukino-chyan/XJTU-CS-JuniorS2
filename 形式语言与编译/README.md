## 形式语言与编译

**授课教师**：赵银亮

**课程内容**：形式语言与自动机+编译原理二合一，课容量非常大。前者主要介绍DFA，NFA，RE，多语言联合DFA，下推自动机。后者介绍词法分析，语法分析（LL1框架、LR0框架，SLR1框架），语义分析（符号表、语法制导的语义分析），中间代码生成（生成三地址代码，涉及算术表达式、布尔表达式、函数调用、控制流语句、数组下标翻译），运行时环境（主要讲栈运行时的栈情况），优化（新加的，只考了寄存器分配）。

**成绩组成**：30%平时成绩（6次作业（要求不能手写），两次大作业（大作业工作量约等于一个编译器专题实验，意义不明））、70%期末。

**考试的一些回忆：**

**判断题**：考察概念，总体不算难，由于被后面三道大题恶心的太严重全忘了。

**大题**（可能有部分小问缺失，但大体考察的点是全的）：

1.考察前四章的内容。题目给出了FLO的RE以及ID的NFA（形式化表示，没有给出图例），有以下几个小问(有两个小问记不太清，但是没有涉及多的知识点)：

(1). 画出FLO和ID的NFA的状态转移图（变相考察RE转NFA）；

(2). 对于两个NFA，将其转化为多语言联合DFA，给出你的转化思路，并完成转化（转化成g-NFA再转DFA，最后转出来好像有6个状态，挺麻烦的）；

(3). 对于一个表达式，用你的多语言联合DFA来进行识别，给出过程和结果。

2.考察5-8章的内容。有以下几个小问：

(1). 给出了一个文法，将其修剪为LL1文法（有左递归需要修剪），并判断修剪后的文法是不是LL1文法（需要你给出详细的判断过程）；

(2). 对于修剪后的文法，填写预测分析表。

(3). 对于源文法，画出其ItemDFA。其中ItemDFA有0-11，共12个状态，题目给出了大致的ItemDFA的图例，要求你补全（实际就是上自己从头画）。比较逆天的是题目要求若有错误自行修改，实际上确实有错误，给出的图例有一个状态你甚至无法抵达。

(4). 写出其SLR1分析表的三行，这三行中有两行有移进/规约冲突，你需要消解这两个冲突。我记得一个冲突SLR1可以消解，一个冲突要用到左/右结合的思想。

3.考察9-12章的内容，这里有30分，非常之逆天。

代码形如（来源：https://xjtu.app/t/topic/12940/13）：

```C++
一些定义
func1(有参数，有函数形参){接受一个函数，并在内部调用}
func2(){
    func3(){好像里面是个if else结构} //定义在func2内
    func1(&func3); //调用func1传入func3
}
func2();
```

(1). 写出func1的词法分析结果。你给出的答案大概就是：(INT,_)(ID,func1)……这种，一个好消息是你不用记住Token种属的名字，题目会给出。

(2). 要求根据主文法，画出func1的语法树。逆天的点是主文法没给全，老师补了两次之后开摆了说让大家根据自己对主文法的记忆与理解画。

(3). 画若干函数的符号表，需要考虑中间代码的临时变量。所以你完成的符号表除了基础的内容，还需要有@code的部分来填写中间代码。之后你还需要对一个函数完成寄存器分配。

(4). 函数的序言、运行过程、尾声的编写。这里一点都没复习，只记得考这些了。

(5). 画栈快照。反正所有的函数参与到了调用的过程中，啥情况都覆盖到了，Chapter11的内容需要全部学会。一个难点是函数双字的填写。
