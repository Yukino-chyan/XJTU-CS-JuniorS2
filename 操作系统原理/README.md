## 操作系统原理

**授课教师**：赵鲲、丁菡、齐赛宇

**课程内容**：课程的授课大纲基本就是408的OS大纲，涉及操作系统结构、进程与线程、CPU调度、进程同步、死锁、内存管理、文件系统、IO系统这些内容，使用的是恐龙书以及配套的英文PPT，看起来很费劲。

**成绩组成**：10%平时成绩（考勤，上课回答问题有加分）、20%作业、70%期末。

**课程评价及经验**：

课程质量尚可，以考研为目标的同学建议好好跟着学一遍，会减轻考研复习OS的压力。

想要学好这门课建议直接跟着王道408的公益课学就完了，基本能覆盖所有的考试内容，也有一部分内容（例：Linux是微内核还是宏内核，MMAP适合什么情况下的文件读取）这种东西会在选择题考到，所以如果想追求高分，也建议过一遍PPT理解这些408课程涉及不到的内容（这些内容不会考简答题，所以理解即可）。

出于一些原因我在大三上没有修读这门课程，刚好这门课程在创新港在下半学期有开课，所以就在下半学期进行了补修。但是两个学期的考试形式上有一些略微的差别。例如上半学期的课程考试是中英双语的，下半学期是全中文考试，可能会考的略微简单一点。但是考试题型都是大差不差，具体的题型如下：

**选择题**：15道，一道两分。重点考察概念的理解，需要死记硬背的部分少。

**简答题**：4道，一道五分。我印象中的题目有：

1.IPC的方式？简单说说不同IPC方式适用的场景。

2.页式存储管理的全过程描述，越详细越好。

3.还有一道题考察文件系统相关知识的，不难。

**计算题**：总体不难，都是经典题型。考察涉及的有：

1.CPU调度：考察了FCFS，SJF，RR这几种经典的算法，你需要计算计算周转时间、等待时间等数据，并且需要理解哪些调度算法会发生饥饿，且要解释原因。

2.银行家算法：考察各种矩阵（Need，Available，Allocation）的计算，考察找出安全序列以及是否为安全状态的判断，考察是否应该对一个Request允许分配资源等。总体考的很简单。

3.内存管理：考察了有TLB的情况下的页式存储管理逻辑地址转物理地址，考察了虚拟地址与物理地址之间的转换，还有页面置换算法相关的题目，考了FIFO、LRU以及CLOCK，其中CLOCK算法考的比较简单，有一个页面访问位和修改位都是0。

4.文件系统：考察计算多级索引文件系统允许存储的单个文件大小的计算。

5.磁盘调度算法：给一个磁盘访问请求序列，要求给出SSTF和C-SCAN两种算法访问的序列，以及计算平均寻道时间

6.信号量的大题考了一个根据前驱图写信号量PV操作，没有考察给一个具体情境来写PV操作的题目。

总体考的不难，老师考前会发复习PPT，一定要好好看。
